# UX Audit — 2026-05-22

Propose-only audit of the top 5 pages by assumed traffic for jessetek.net. No code or markup was changed. Findings come from a static review of the HTML in `public/` plus verified greps; GA4 / Microsoft Clarity were not directly queryable in this session (auth required), so traffic ranking is a best-guess based on the site's primary funnel.

Top-5 traffic guess (in priority order):
1. `/home`
2. `/buyer`
3. `/seller`
4. `/valuation`
5. `/rates`

---

## 1. /home (`public/home.html`)

- **Friction:** 3 in-page CTAs use `href="#contact"`, but the page has no `id="contact"` anchor (grep-confirmed: 3 hrefs, 0 ids). Clicks scroll to the top of the page instead of a contact CTA — silent UX dead-end with no error, easy to miss in QA.
  - **Fix:** Either add `id="contact"` to the existing strategy-call CTA section, or rewrite the 3 hrefs to point at the actual CTA section's id (or to `/zoom`).
  - **Confidence:** high (verified in code).

- **Friction:** Step-cards are large `<a>` blocks with photo + overlay + text + iMessage proof bubbles. On a 1546-line page, hover/focus states for these are critical for affordance, but no `:hover` / `:focus-visible` confirmation was done in CSS during this audit.
  - **Fix:** Audit `public/css/main.css` for `.step-card:hover` and `.step-card:focus-visible` outline/elevation styles; add if missing so keyboard users get a clear focus ring.
  - **Confidence:** medium (likely — common on hand-styled card grids).

- **Friction:** Page is 1546 lines. If the primary "Book a Strategy Call" CTA lives mid/late in the doc, mobile users may scroll past it without seeing it pinned.
  - **Fix:** Verify `cro.js` is mounting a sticky mobile CTA bar on `/home`; if not, add one bound to the same destination as the broken `#contact` links.
  - **Confidence:** medium.

## 2. /buyer (`public/buyer.html`)

- **Friction:** 11 step-card links point to `href="#contact"` and there is no `id="contact"` on the page (grep-confirmed). Every step in the buyer journey ends in a click that scrolls back to the hero — a serious dead-end on the highest-intent page.
  - **Fix:** Add a `<section id="contact">` wrapping the existing strategy-call CTA, OR change all 11 hrefs to `/zoom` (Jesse's Zoom booking page) or `#cta`. Single-line fix, biggest expected lift on this page.
  - **Confidence:** high (verified — 11 occurrences in code).

- **Friction:** Live-rate strip (`#liveRateBanner`, ~line 119) renders before its data fetch resolves. If `/api/rates` is slow or fails, the user sees a blank/placeholder strip with no fallback copy.
  - **Fix:** Inline default copy (e.g. "Today's rates loading…") inside `#liveRateBanner`, and on fetch failure swap to "Call Jesse for today's rate" instead of leaving empty.
  - **Confidence:** medium (typical async-render pattern — needs runtime confirmation).

## 3. /seller (`public/seller.html`)

- **Friction:** Same 11 broken `#contact` anchors as buyer.html (grep-confirmed: 11 hrefs, 0 ids). Sellers — high-LTV leads — have no working in-page CTA on the step cards.
  - **Fix:** Same as buyer — add `id="contact"` to the existing strategy-call CTA, or repoint to `/zoom`. Coordinate the fix across buyer + seller in one pass.
  - **Confidence:** high (verified).

- **Friction:** Secondary lead magnet CTA points to `/guide` (the buyer guide PDF) on a seller-focused page, which is a persona mismatch.
  - **Fix:** Swap the secondary CTA on `/seller` to `/seller-checklist` or `/seller-net` (net-proceeds calculator) — both already exist in `public/`.
  - **Confidence:** medium (needs visual confirmation of which exact button — link target was flagged in subagent review).

## 4. /valuation (`public/valuation.html`)

- **Friction:** Multi-step valuation form (~lines 142–181) has no visible `aria-live` region for inline validation errors. If a step's validation fails, screen-reader users and many sighted users won't know what went wrong.
  - **Fix:** Add `<div role="alert" aria-live="polite">` near the form's submit button and write validation errors into it on failed step transitions.
  - **Confidence:** medium.

- **Friction:** Exit-intent modal copy is hardcoded for a buyer-focused "5 Mistakes" guide on a page whose audience is sellers asking for a home valuation — wrong-funnel offer at the highest-intent moment.
  - **Fix:** Swap the exit-intent offer on `/valuation` to a seller-relevant lead magnet (e.g. "Free Seller Checklist" → `/seller-checklist`, or "What's your home really worth?" recap PDF).
  - **Confidence:** medium (high-impact if confirmed, hence worth verifying first).

## 5. /rates (`public/rates.html`)

- **Friction:** TCPA / consent disclosure is buried inside a `<details>` collapsible — most users will never expand it before submitting. Beyond UX friction, this is a TCPA compliance risk: opt-in language must be conspicuously displayed, not hidden.
  - **Fix:** Surface the TCPA copy as always-visible small text directly above the submit button, with a required `<input type="checkbox">` opt-in. Don't hide consent behind a disclosure.
  - **Confidence:** high (clear pattern flag in the markup; also legal risk, not just CRO).

- **Friction:** LeadConnector iframe (~line 177) has a hardcoded `height="460"`. On narrow viewports the form content stacks vertically and overflows, producing a nested scrollbar inside the iframe — a known mobile friction pattern.
  - **Fix:** Either drop the fixed height and let the iframe auto-size via `postMessage` height events (LeadConnector supports this), or set `min-height: 460px; height: auto;` and add a `loading="lazy"` for below-fold.
  - **Confidence:** medium.

- **Friction:** Rate chart timeframe toggle buttons (~line 140) swap data via fetch but show no loading state. Users may think the toggle is broken on slow connections.
  - **Fix:** Add `aria-busy="true"` + a small inline spinner / disabled state on the active button during fetch.
  - **Confidence:** low (hunch — depends on chart JS behavior not reviewed here).

---

## Cross-cutting themes

- **Broken `#contact` anchors are the single highest-confidence issue** across home/buyer/seller (25 dead links across the three pages). One coordinated fix.
- **Wrong-persona lead magnets** show up on both /seller (buyer guide CTA) and /valuation (buyer exit-intent). Worth a single sweep of secondary CTAs on seller-funnel pages.
- **Async data with no fallback copy** appears on /buyer, /seller, and /rates — add a small "loading…" + failure-state pattern once and reuse.

## Method / caveats

- Traffic ranking was assumed, not pulled from GA4. Microsoft Clarity (`clarity.microsoft.com/projects/view/wke2elyrvc`) was not accessible in this session.
- Findings tagged `high` were verified by grep against the actual HTML. `medium` / `low` items were flagged from code patterns and would benefit from Clarity heatmap / GA4 event confirmation before code changes.
- No HTML/CSS/JS was modified. This is a propose-only audit.
